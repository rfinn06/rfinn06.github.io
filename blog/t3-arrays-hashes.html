Array and Hashes

8/16/15

What is an Array and What is a Hash

Arrays and Hashes are both a collection of objects in Ruby. These collections are meant to store data in an organized manner so that we can easily grab or call on this data quickly at any time. As an example, let's imagine we are creating a program that stores and manipulates data on baseball players. Let's say we have a collection of batting averages and want to create a new list of this data in an organized manner. I will illustrate this example using an array.

Using Arrays

As we noted above, an array is a collection of objects. In Ruby, an array looks like this: 

batting_averages = [0.225, 0.123, 0.389, 0.212]

The above example is our list of batting averages. Note that we first have the name of our array, batting_averges, and our objects are listed inside of square brackets and separated by a comma. We can also have a list or collection of strings in arrays as well. A string gets listed inside an array just like integers or floats, except that a string has to have quotation marks around it. Also note that becuase we are using floats, or numbers with a decimal point, we put "0" at the beginning of our values. Ok, now we have our array of batting averages it's now time to sort this list.

In Ruby there are built in ways or methods to manipulate objects and arrays. One of them is called "sort". Pretty self-explanatory, eh? To sort our list of batting averages we do the following:

batting_averages.sort

We applied the sort method to our array. This will produce a new array that is sorted:

[0.123, 0.212, 0.225, 0.389]

You can learn more about arrays and their methods by visiting the Ruby Docs.

Using Hashes

We can use an array in Ruby to store batting averages and use different methods to sort or manipluate that data. What if we wanted to have a list of not just batting averages, but of individual players' batting averages. This is where hashes come in. An array organizes or indexes its objects by numerical order. In other words, it says that the first object listed in the array is object one, the second lsited is object two, and so on. Hashes on the other hand do not organize their objects in a strict numberical order. In fact, hashes are not ordered at all. Since objects are not indexed by a numerical value we use keys and assigne them a value. So a hash looks like this:

player_batting_averages { "Babe Ruth" => 0.345
                          "Frank Thomas" => 0.325 }

A hash consists of keys that have values. To assign a value to a key we use the "=>" operator. In the example above the players are the key and the batting averages are the values. As with arrays, there are many things you can wdo with hashes. You can pull values from a hash by using the key, you can create a new array of values from a hash and much more. You can explore more about hashes and what you can do with them by visiting the Ruby Docs.

Summary

  • Arrays and Hashes are used to build a list of object in Ruby.
  • Arrays are an ordered list of objects that are numerically indexed.
  • Hashes are an unordered lsit of objects that are indexed by keys
  • We can do many things with arrays and hashes un Ruby
